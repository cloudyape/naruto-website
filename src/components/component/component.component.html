Components are like Blocks. Join them to make Views

In Naruto, a component is a fundamental building block that encapsulates the behavior and presentation of a part of the user interface (UI). Components are reusable, modular, and self-contained units that help organize the structure of a Naruto application.

Creating New Component

  • Run command -> naruto n c componentName, to create a new compoent. For Eg. naruto n c newComponent

Call a component inside another Component

  • Simply add tag of the component in HTML. Eg <newComponent>
  • Once the component is Added go to JS and write -> loadTag('componentName'). For Eg. loadTag('newComponent')

Component Data Binding with Real Time Updation

  • Create a global Variable in JS using window.variableName and Define Value. For Example window.yeah = "Hello Naruto"
  • Now Call the variable in html using -> ${variableName}. For Eg ${yeah}
  • To Update it realtime call function -> updateVariable('variableName', 'Variable New Value'). For Eg updateVariable('yeah', 'Hello Sasuke')

Component Routing

  • Just update Component name and path in routing.json file. For Example
    {"routes": [ {"path": "/", "component": "app/app.component.html"}, {"path": "/gettingStarted", "component": "gettingstarted/gettingstarted.component.html"}, {"path": "/component", "component": "component/component.component.html"} ] }
